# Map Exploration Audit

## 1. Scope
Аудит выполнен только по:
- `js/map.js`
- `js/ui.js`
- `index.html`
- `css/style.css` (только влияние на интеракции)
- `data/features.geojson`

Оценивались сценарии: загрузка карты, отображение объектов, выбор/переход между объектами, timeline, взаимодействие верхних/нижних панелей.

## 2. Confirmed working behavior
- Карта инициализируется, GeoJSON загружается, валидируются только `Point`-объекты, есть bootstrap-сигналы готовности/ошибки.
- Выбор объекта работает через клик по маркеру/карточке: выделение на карте, фокус на объект, открытие detail panel.
- Hover-связка карта ↔ карточки работает (подсветка при наведении).
- Timeline влияет на фильтрацию карты и списка через единый year-filter.
- Верхние панели (`Filters`, `Layers`, `Bookmarks`, `Courses`, `LIVE`) открываются/закрываются централизованно.

## 3. Issues
1. **Стартовый UX скрывает почти все данные при первом входе**
   - Timeline в `point`-режиме по умолчанию ставится в `max year` набора.
   - Для текущего `data/features.geojson` диапазон `-3100…2010`, и на точке `2010` виден только `1/19` объект.
   - Пользователь стартует в состоянии «данных почти нет», хотя данные есть.

2. **Поиск не поддерживает исследование при множественных совпадениях**
   - `buildSearchResults` возвращает только первые 5 результатов без ранжирования/сортировки.
   - При широком запросе пользователь не может дойти до остальных совпадений через dropdown.

3. **Timeline фактически выключен из клавиатурного сценария**
   - Нативные range-input принудительно исключаются: `pointer-events: none` + `tabIndex = -1`.
   - Управление оставлено на кастомные pointer-жесты; keyboard-путь для исследования времени отсутствует.

4. **Навигация по объектам ограничена первым окном списка**
   - Лента карточек рендерит только первые 80 объектов (`slice(0, 80)`) без механики перехода к остальным.
   - При росте датасета пользователь теряет доступ к части результатов через основной list-flow.

5. **Риск визуальной перегрузки карты в среднем объёме данных**
   - Кластеризация включается только после `>500` features.
   - До этого порога точки всегда отрисовываются поштучно; при плотных данных это создаёт overlap и ухудшает findability.

## 4. Impact classification
- **BLOCKER**
  - Стартовый timeline в точке `max year` (видимость 1/19 на старте).

- **HIGH**
  - Поиск ограничен 5 результатами.
  - Timeline недоступен в keyboard-flow.

- **MEDIUM**
  - Лента карточек ограничена 80 элементами без продолжения.
  - Кластеризация только после 500 (перегрузка в среднем диапазоне).

- **LOW**
  - Не выявлено.

## 5. Recommended next step
Сделать короткий remediation-plan (без редизайна):
1) зафиксировать baseline exploration-flow (first meaningful result ≤ 2 шага),
2) закрыть BLOCKER,
3) закрыть HIGH в одном UX-пакете,
4) затем проверить MEDIUM на реальном объёме данных.
