# Map Search Audit

## 1. Scope
Проверено в рамках:
- `js/ui.js`
- `js/map.js` (поиск → выбор объекта → фокус карты)
- `index.html` (поисковый input/dropdown/no-results)
- `css/style.css` (только влияние на search-flow)

## 2. Confirmed working behavior
- Поиск запускается по `input` с debounce и работает в текущем exploration state.
- Search results строятся из актуального `filteredFeatures` (учитывают активные ограничения).
- Выбор результата переводит фокус на объект на карте и открывает detail panel.
- No-results состояние показывается, есть действие очистки поиска.
- Очистка поиска корректно сбрасывает search-state и обновляет UI.

## 3. Issues
1. **HIGH** — Жёсткий лимит выдачи: только 5 результатов.
   - При множественных совпадениях пользователь не может просмотреть остальные результаты через search dropdown.
   - Это напрямую ограничивает discovery при широких запросах.

2. **MEDIUM** — Нет видимого сигнала, что выдача усечена.
   - Пользователь видит до 5 элементов без индикации «показаны не все».
   - При больших совпадениях это воспринимается как полный список, хотя это не так.

## 4. Final status
**STABLE WITH MINOR GAPS**

## 5. Recommended next step
Сделать один минимальный шаг: поднять cap search dropdown с 5 до безопасного значения (например 12) с сохранением текущего flow.
